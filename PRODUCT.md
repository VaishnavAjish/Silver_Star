# Product

## Register

product

## Users

Factory operations staff — inventory operators, process workers, and supervisors on the production floor of a diamond/gemstone manufacturing facility. They work with physical lot traceability, machine process tracking, weight/dimension measurements, and purchase flow. They are task-focused, work quickly, and expect the tool to stay out of their way.

## Product Purpose

Silverstar Grow ERP is an internal manufacturing operations system for tracking diamond lot lifecycles: from raw rough to grown/cut stone. It manages inventory movement, process assignments, growth run cycles, purchase orders, and financial records. Success means zero ambiguity about where a lot is, what happened to it, and what its current measurements are.

## Brand Personality

Professional, trustworthy, calm. The interface should feel like a well-maintained precision instrument — reliable, dense, legible, never surprising. Operators should feel confident in the data they're reading. Nothing flashy; every element earns its place.

## Anti-references

- Generic startup SaaS (Notion, Linear) — over-minimal, too much whitespace, wrong register for a factory tool
- Consumer lifestyle apps — wrong emotional tone entirely
- Old ERP software (SAP, Tally) — dated, cluttered, looks like 2005

## Design Principles

1. **The tool disappears into the task.** Operators are reading lot codes and weights, not admiring the UI. Density is a feature; empty whitespace is waste.
2. **State is never ambiguous.** Every element has a clear default, hover, focus, active, disabled, and error state. If it can be interacted with, it shows it.
3. **Consistency over surprise.** Same button shape, same form vocabulary, same icon weight across every screen. Delight is a moment, not a page.
4. **Motion earns its place.** Transitions convey state change and feedback only. No choreography, no orchestration.
5. **Brand through restraint.** The forest green carries authority precisely because it is used only for primary actions, active states, and key indicators — never decoration.

## Accessibility & Inclusion

WCAG 2.1 AA minimum. Keyboard navigation required for all interactive elements. Focus rings must be visible and high-contrast. No reliance on color alone to convey state.
